В данном репозитории представлена игра Heroes. Проект посвящён разработке алгоритмов для пошаговой стратегии с боевой системой. 
Реализованы ключевые алгоритмы, которые отвечают за генерацию армии противника, симуляцию пошагового боя, определение доступных целей для атаки и поиск кратчайшего пути между юнитами.

В проекте реализованы **4 обязательных алгоритма**, используемых игрой через подстановку JAR-файла.

1. Генерация армии противника
**Класс:** `GeneratePresetImpl`

**Описание:**  
Формирует армию компьютера, оптимально используя доступные очки (`maxPoints`).  
Учитываются ограничения:
- не более **11 юнитов каждого типа**;
- суммарная стоимость армии не превышает `maxPoints`.

**Алгоритм:**  
Greedy algorithm:
1. Сортировка шаблонов юнитов по коэффициенту эффективности  
   `(baseAttack + health) / cost`
2. Последовательное добавление юнитов, пока хватает очков.

**Сложность:**
- сортировка: `O(n log n)`
- формирование армии: `O(n)`
- **итог:** `O(n log n)`

 2. Симуляция боя
**Класс:** `SimulateBattleImpl`

**Описание:**  
Пошаговая симуляция боя между армиями игрока и компьютера.

**Алгоритм:**
- бой проходит раундами;
- в каждом раунде юниты ходят по убыванию атаки;
- каждый юнит выполняет `unit.getProgram().attack()`;
- погибшие юниты исключаются из дальнейших ходов;
- бой завершается, когда одна из армий полностью уничтожена.

**Сложность:**  
`O(n × m)`, где
- `n` — количество юнитов игрока
- `m` — количество юнитов компьютера

3. Отбор юнитов, доступных для атаки
**Класс:** `SuitableForAttackUnitsFinderImpl`

**Описание:**  
Определяет, какие юниты могут быть атакованы:
- для левой армии — если слева нет союзника;
- для правой армии — если справа нет союзника.

**Алгоритм:**  
Линейный проход по рядам с проверкой соседних координат.

**Сложность:**  
`O(n)` — фактически линейная, так как количество рядов фиксировано.

4. Поиск кратчайшего пути между юнитами
**Класс:** `UnitTargetPathFinderImpl`

**Описание:**  
Находит кратчайший путь от атакующего юнита к цели с учётом:
- границ игрового поля (27 × 21);
- занятых клеток (другие юниты).

**Алгоритм:**  
Поиск в ширину (BFS) по клеткам игрового поля.

**Сложность:**  
`O(W × H)`, где
- `W = 27` — ширина поля
- `H = 21` — высота поля

## Запуск игры

Запустить игру командой:

```bash
java --enable-native-access=ALL-UNNAMED -jar "Heroes Battle-1.0.0.jar"
В игре:

нажать «Начать новую игру»

создать армию игрока

сгенерировать армию компьютера

начать бой

Игра автоматически использует реализованные алгоритмы из obf.jar.

Используемые технологии
Java 17+

IntelliJ IDEA

Git / GitHub

BFS, жадные алгоритмы, сортировка, множества, очереди
